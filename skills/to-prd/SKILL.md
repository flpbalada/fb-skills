# Write PRD From Conversation

Read the full conversation thread first.

Understand:

- Raw requirements
- Context
- Constraints
- Open questions
- Implied user goals

Then convert the requirements into a concise PRD.

## Input

- User-provided topic, request, transcript, notes, or argument text.

## Output

- Problem or goal
- Users or actors
- Requirements
- Non-goals
- Constraints
- Open questions
- Acceptance criteria
